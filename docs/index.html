@com-junkawasaki/office-style · style-ir/1

pptx から
トーン・マナー・スタイルを抽出

アップロードした .pptx を、配色・フォント・スライドマスター・ガイドの確定値と、Gemma 4 vision が読む印象に分けて抽出し、単一の StyleIR に正規化する TypeScript パッケージ。

使い方 (README) アーキテクチャ StyleIR スキーマ

2 レイヤを 1 つの IR に

構造は OOXML から確実に、印象は vision から。役割を分けて重複なく相補的に。

決定論レイヤ 誤差ゼロ

theme clrScheme 12 色・fontScheme、slideMaster/Layout の placeholder と既定テキストスタイル、clrMap 解決、ガイド (1/8pt→EMU 校正)、複数 master/theme を OOXML から直読み。

解釈レイヤ ローカル LLM

レンダリングしたスライド画像を Gemma 4 vision に通し、フォーマリティ・ムード・カラームード・密度・ビジュアルスタイル・要約を ToneProfile として推定 (confidence 付)。

パイプライン

抽出 → 描画 → ラスタライズ → vision → 集約。下流は単一 IR だけ読めばよい。

.pptx ─► extractStyle ─► StyleIR ──► toSvgraphPresentation ─► svgraph editor │ (masters / layouts / guides / text_styles) ├─ renderDeckSlides(実スライド) ┐ └─ renderPreviewSvg(代表1枚) ───┴► SVG ─► raster ─► PNG ─► Gemma vision ─► ToneProfile (resvg/canvas) (cpu·q4 / webgpu·q4f16) │ aggregateTones ◄─ 複数スライド集約 ◄──────────┘

主な機能

slide master 対応・複数テーマ・実機 vision・svgraph 流し込み。

🎨 デザイントークン

配色 12 色・major/minor フォント・スライドサイズを正確に抽出。

🧩 slide master 継承

placeholder のフォント/サイズ/色を master の txStyles から継承して解決。

🗂 複数 master/theme

slide→layout→master の継承チェーンで各スライドを正しいテーマに束ねる。

🖼 内蔵レンダラ

図形・グラデ・回転・表・画像・色解決を SVG 化して vision 入力に。

🤖 Gemma 4 vision

ローカル・API キー不要。Node(cpu) / ブラウザ(WebGPU) 両対応。

🔗 svgraph 流し込み

presentation IR (masters/guides/text_styles) を svgraph editor へ。

使ってみる

CLI

# デザイントークン要約
office-style tokens deck.pptx

# StyleIR を JSON 出力
office-style extract deck.pptx

# 実スライド → vision でデッキトーン
office-style tone deck.pptx --deck

# svgraph presentation IR へ
office-style svgraph deck.pptx

API / ブラウザ

// Node
import { extractStyle } from
  "@com-junkawasaki/office-style";
const ir = extractStyle(bytes, "deck.pptx");
ir.theme.colorScheme.accent1.hex; // "#3B82F6"

// ブラウザ (WebGPU + q4f16)
import { analyzeDeck } from
  "@com-junkawasaki/office-style/browser";
const { ir, perSlide } = await analyzeDeck(bytes);
